② 8 个关键设计决策

决策点决定理由
主标题 "今天,我能帮你做什么?" 设问句,让用户立刻知道这是工具不是查阅
能力区数量 v1 = 2 个(知识问答 / 文档处理) 5 类里只 2 类落 v1;其他暂不显示,避免 dead button
能力区命名 "问知识 / 处理文档" 极口语,回避"问答 / 智能"等术语(P1 零技术词)
"最近"列表 按 task instance 倒序,3 类形态混排 用户记忆点是"我做过什么",不是"哪种类型"
"部门提醒" self-improving 在用户端的小窗口 用户感受到"系统在帮整个部门进步";仅 1 行,无红点
是否有侧边栏 没有 v1 试点单部门,导航极简就够;侧边栏会让首屏复杂
顶部右侧 用户名 + 部门 让用户始终知道自己的"上下文"
设置入口 点头像 → 下拉菜单(个人记忆 / 退出) 不让设置抢首屏注意力

③ "部门提醒"是 self-improving 的用户端出口

显示规则 过去 7 天部门有 ≥ 5 次同主题查询 + 当前用户没问过该主题 → 显示。点击预填该主题进能力区,不强制。
不能做什么 不能变成"团队广播 / 集体讨论 / 评论"——那是飞书的事。这里只是 Insight 的轻量出口。
视觉权重 浅黄底 / 1 行 / 无红点徽章 / 不打扰。和系统提示性 banner 同级,不抢能力区。
价值 让员工感受到"AI 在背后看着部门趋势" + "原来同事也在关心这个"。这是 self-improving 在用户端最细微的体感。

④ 5 条主路径设计原则(贯穿 16/17/18 三屏)

#原则含义
P1 零"任务 / 配置 / Agent / Skill"字眼 UI 文案严禁出现技术词。用户从来不点"创建/新建/配置",只点能力区和"问/上传/继续"
P2 第一动作即创建,空状态不入库 用户点能力区只是"进入",不创建任何 Ledger。第一个真实动作(敲句话/上传文件)才创建 TaskInstance
P3 永远显示来源 / 引用 / 置信度 知识问答的来源卡 + 文档智能的字段引用 + 置信度三色 — 可追溯不变量的物理体现
P4 诚实失败用主动语态,给具体下一步 不用"抱歉"挡住,给具体行动选项("换个表述 / 告诉 Lead 这块缺资料")
P5 个人/部门 memory 透明但不打扰 永远不主动 prompt"要不要记住";用户主动写,系统在每次问答下方一行小字告知"基于...",可展开看具体注入

这套 UX 让张三从来不需要思考:
❌ "我用哪个 KB?"(KB chip 默认全开)
❌ "我用哪个 agent?"(用户层根本不知道有 agent 概念)
❌ "怎么配置任务?"(零配置 / 直接干活)
❌ "怎么让 AI 记住我?"(透明栏告诉他记住了什么 / 一键编辑)
