② 8 个关键设计决策

决策点决定理由
看板布局 三段:用量趋势 / 内容洞察 / 行动入口 从"全貌"到"具体"递进 · 符合阅读习惯
时间窗切换 7 / 30 / 90 天三段(默认 30 天) Ops Lead 关注的运营周期
任务结构 横向条形图(不饼图) 任务类型 ≤ 5 类 · 条形图更易读且能横向对比
高频主题 top 5 + "查看全部"展开 5 秒决策,不超载
bad case 卡 按"失败模式"聚合(不是按时间) admin 关心"什么失败模式高频",不是"昨天哪条失败"
提议卡 显示数量 + top 3 摘要 + 单条 [审] 按钮 + 全部入口 让 admin 在看板就能处理简单提议,复杂的进 inbox
记忆健康度 三档着色,低命中直接给"查看"建议 self-improving 的体感;让 admin 看到"哪些规则该清"
不显示 用户排行榜 / 工时统计 / 单 query 原文 / 模型成本 / token / 错误率 隐私边界(PA2) + 双投影 PA4(成本/错误率是 global admin 的事)

④ 屏 3(看板) ↔ 屏 6(提议箱)的衔接

双层模式:屏 3 主屏的"💡 系统提议 (3 待处理)"卡是屏 6 的快捷入口。

• 屏 3 只显示 top 3,每条带 [审 →] 直接进屏 6 单条详情
• "全部提议 →"进屏 6 完整 inbox

 这样 admin 90% 时间在屏 3 一眼扫过,紧急的 3 条直接处理;周一 / 月初专门进屏 6 集中处理积压的。

⑤ 看板横切设计点

点设计
数据刷新按小时(用量趋势 / bad case 增量) + 按天(高频主题聚类 / 提议)
空状态"还在收集数据" + 引导:"请部门成员开始使用工作台"
数据导出不提供(隐私边界 PA2)
异常颜色灰 / 蓝 / 黄 / 红 四档 · 红只用于"必须立即看的" bad case
移动端v1 不做(admin 看板桌面端为主)
