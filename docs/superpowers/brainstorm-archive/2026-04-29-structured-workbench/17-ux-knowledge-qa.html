③ Memory 透明栏 — 透明但不打扰

设计哲学:用户想知道"AI 是基于什么回答我的",但又不想被技术细节淹没。

默认折叠(1 行):告诉用户有"角色 + N 部门规则"在生效。
点击展开:具体显示注入的 facts。
"编辑我的记忆 →":从聊天面直接进 /profile/memory — 让用户在聊天中发现"AI 不知道我的角色",一键改,下次就懂。

 这是"AI 越用越懂我"的反馈回路 — 不是单向注入,是用户能感知 + 调整的闭环。

失败 4 · 越权命中(不应出现)

⚠ 系统拦截 + 自动告警 — 用户感知不到失败原因,但不会拿到不该看的内容。
反对模式:❌ "抱歉,可能没有理解您的问题..." — 客气话挡住,用户不知道下一步该怎么办。
正确模式:✅ 主动语态 + 给出具体行动(换说法 / 告诉 Lead / 看原文 / 继续生成)。

⑤ "告诉 Lead 这块缺资料" — self-improving 在用户端的另一个出口

价值:用户的一次"找不到"自动变成 dept admin 的一条 KB 补充提议。

点击按钮 → 系统自动构造一条 KB 补充候选(F3)写入 t_proposal_inbox · target = dept_admin · 证据 = 当前 query + 用户 + 时间。
用户得到反馈:"已通知 Lead 这块需要补资料,等资料补上后会通知你"。

 这是用户端最强的 self-improving 信号通道 — 业务的痛点直接进入产品进化通道,不需要再"提需求 / 走流程"。
