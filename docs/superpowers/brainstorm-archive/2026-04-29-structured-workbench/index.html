📚 完整目录(按叙事章节)

章 1 困境与立意 · why 为什么不走老路?
01
产品形态选择
KB 中心化 vs Chatbot 集合 vs 结构化工作台 — 三种形态对比,最终选 C。
 03
PaaS 作坊 vs 工作台
为什么不是 agent 拖拽平台 — 复用单位错了,不是工具不好。
 08
场景五分法 · 智能工厂
按工作模式切片,用少数几类 Agent 覆盖 80% 任务 — 4 类主干 + 1 类兜底。
章 2 架构骨架 · what 三层抽象怎么切?
02
Task vs Agent+Skill
用户层是任务,系统层是 Agent+Skill — 两层都要,各司其职。
 06
UX 术语锁定
Task 是后端 entity 不是 UI 概念 — 用户看到的是"知识问答 / 文档处理"。
章 3 v1 范围与治理 · how(工程版) 试点要做多少?怎么治理?
05
v1 范围 + 实施路径
3 种实施路径对比,最终选 A:渐进改造(试点期保留新旧并存)。
 10
v1 / v2 / v3 切分
2 实现 + 3 预留 + 1 兜底 · v1 拆 9 个 PR · 6-8 周完成。
 11
试点权限取舍
90% 传统权限可省,但 G7 信任契约不能漏 — 不重不漏。
 14
双级 admin 视角分割
系统医生 vs 业务管家 — 同一份 Ledger 双投影,看完全不同的世界。
章 4 Self-improving 闭环 产品怎么自己长大?
04
持续学习横切层
Ledger 为底盘的 self-improving + 部门 insight 闭环。
 12
三件事关系
User Memory / Dept Memory / Dept Insight — 试点期持续学习的核心三件事。
 13
Self-improving 发动机
5 种提议机制 + 反学习 + 业务永远只面对二元决策。
章 5 看见产品 · see 用户会看到什么?
16
UX 屏 1 · 工作台首屏
能力区 + 最近 + 部门提醒 · 用户进入即可干活,无技术词。
 17
UX 屏 2a · 知识问答
KB chip + Memory 透明栏 + 来源卡 + 4 种诚实失败 + "告诉 Lead"出口。
 18
UX 屏 2b · 文档处理
协议查询(NL)+ 协议提取(上传) · 字段卡 + 6 种失败 UX。
 19
UX 屏 3 · Insight 看板
业务管家的眼睛 — 用量 / 主题 / bad case / 提议 / Memory 健康度。
 20
UX 屏 6 · 系统提议 inbox
三段式提议卡 + 反学习二次提议 + 不能"全部批准"边界。
终幕 深潜与闭环 工程承载 + 一图收束
09
复杂指令 + 问题排查架构
五分法另两类的工程骨架 — v1 接口预留 / v2 接入。
 15
Ledger 双层 schema
通用强类型列 + 分型 payload — 一张表装下 5 类形态,v1 一次设计 v2/v3 不重构。
 21
整体闭环 · finale
五分法 + 三件事 + 双级 admin + Self-improving 收束 · 10 条不变量 + 术语映射。
关键决策落点(spec 中继续展开)
  • 三层架构:用户层(Task)+ Agent 层 + 底座 + 横切持续学习层
  • v1 范围:2 个 Agent 实现 + 3 个预留 + 1 个兜底 · 双级 admin 雏形 · Memory + Insight + 3 类规则提议
  • 实施路径 A:新增 /workbench 入口,老 /spaces 兜底,试点期可比对
  • UX 术语:UI 上零"任务"二字 · TaskInstance / TaskTemplate 仅存活在后端代码
  • 10 条不变量(I1-I10)跨 v1/v2/v3 都不能违反 — 见 21 整体闭环 完整模式
