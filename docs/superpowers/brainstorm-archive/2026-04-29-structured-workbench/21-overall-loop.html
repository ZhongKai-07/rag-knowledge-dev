③ 四条对外价值主张

🚫

反作坊 — 语料零重复 / 权限零旁路

所有任务共享底座 KB,永不复制上传。任务自动继承 RBAC + security_level。Agent 升级所有任务受益,零迁移。

🧠

AI 越用越懂部门 + 越用越懂我

个人记忆(隐私私有) · 部门记忆(admin 维护)分层注入。Promote 流程让个人沉淀部门资产。

🌱

产品形态自我进化(self-improving)

Ledger 信号 → 5 类提议 → admin 一键决策 → 写回。被拒绝的提议也是反馈,系统也在学。

🤝

诚实失败 / 永远可追溯

知识问答的来源卡 · 文档智能的字段引用 · 失败时主动语态给具体下一步,不用客气话挡住。

④ v1 试点 → v3 完整版的演进路径

能力 v1(Ops 试点) v2(扩部门) v3(全员)
Agent 数量 2 实现(KQ + DI) + Troubleshoot(IT 自助) + ToolOrchestration
权限 双级 admin / dept 内部默认全开 多级 admin / 工具风险分级 跨部门授权 / OPA / OpenFGA
持续学习 用户主动声明 Memory · 规则提议 + Autodream 自动抽取 · 集体记忆 + 反哺 KB(LLM-wiki) · Pulse 推送
UX 入口 /workbench 主路径 + /spaces 兜底 /spaces 下线 多端(移动 / IM 集成)
评测 status + 行为信号筛 70% bad case + thumbs-up/down · 字段纠正信号 + 跨任务 / 跨文档 insight

v1 已经能跑出最小可用闭环。v2 / v3 的增量都是"接已留好的接口" — 不需要重构。

⑤ 跨阶段不变量 — v1/v2/v3 都不能违反

#不变量含义
I1UI 零技术词用户可见区域永不出现 task / agent / skill / prompt / schema 字眼
I2语料零重复所有任务共享底座 KB,永不为某一个 Agent / 任务复制语料
I3权限零旁路所有 Agent 执行走现有 RBAC + security_level + metadata filter,异步路径不读 UserContext
I4可追溯所有 Agent 产出必须能链回原始来源(知识问答→chunk · 文档智能→原文锚点 · 复杂指令→工具回执 · 问题排查→证据链)
I5诚实失败所有 Agent 失败时不能伪装成功(检索 0 → "未找到" · 字段空 → confidence=0,不杜撰)
I6Agent ≤ 5Agent 数量超过 5 = 切片角度错了,需要重新归纳分类法,不是再加一个
I7User Memory 严格私有即使 global_admin 也不能直接看 User Memory 内容
I8Insight 双投影不混合技术投影(global) vs 业务投影(dept) 永远不混合视图
I9每条提议都要有证据 + 影响不许做"全部批准";admin 必须看清证据/影响才能拍板
I10反学习记录拒绝原因提议被拒不是死路 · review_note 必填 · 下次再生提议必须附"上次被拒原因"

⑦ 一句话总结

结构化工作台 = "5 类工作模式骨架 + 双级 admin 治理 + 持续学习自我迭代 + 业务零技术暴露" 的智能操作中枢。

它不是一个工具,是一个会自己长大的产品形态。
Ops 试点 = 最小可用闭环;扩部门 / 全员 = 接已留好的接口。
所有关键决策在前 8 个 brainstorm 文档里逐一锁定,下一步是从 spec 修正(M1-M13) → 实施 plan → 渐进 PR 落地。
