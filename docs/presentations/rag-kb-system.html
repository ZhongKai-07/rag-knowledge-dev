通道 A · 意图导引检索

根据意图分类结果，在对应意图节点的向量分区内精准检索。高置信度时直接给出结果；低置信度触发澄清引导。

通道 B · 全局向量检索

不经意图过滤，对整个知识库做全量语义搜索，作为兜底通道，与通道 A 结果合并后统一重排。

对话记忆上下文注入 全链路 RagTraceContext 多模型调度 + 健康检查

HT KnowledgeBase向量检索
05 · Vector Search

OpenSearch 2.18 · 生产向量存储

每个知识库独立索引，HNSW 近似近邻 + BM25 文本检索双路融合

索引架构

One Index Per KB

每个知识库（collection_name）对应一个独立的 OpenSearch index，天然实现数据隔离。

字段结构

content 文本 + embedding 向量（1536d） + metadata（kb_id, doc_id, chunk_idx, security_level）

检索策略

HNSW 向量检索

基于 Embedding 余弦相似度，M=16 / efConstruction=512，top-K 语义召回。

BM25 + 混合融合

关键词精确匹配与向量语义并行，RRF（倒数排名融合）合并两路结果后送 Reranker。

生产唯一向量引擎 OpenSearch 2.18 k-NN plugin HNSW Docker Compose 部署 软删除复用 collection_name

HT KnowledgeBasePrompt Engineering
06 · Prompt Engineering

提示词工程：检索增强 + 意图感知

Prompt 分三层构建，不同意图分支使用差异化指令，保障答案质量

三层 Prompt 结构

① 系统人设层

定义 AI 角色定位、回答风格、边界约束（"仅基于知识库内容回答，不编造"）。

② 检索增强层

将 Rerank 后的 top-K chunks 结构化注入，标注来源 KB、文档名，支持引用溯源。

③ 对话历史层

按 token 预算注入最近 N 轮对话，维持多轮语境连贯性。

Query Rewriting 策略

意图感知改写

根据意图分类结果，将口语化问题改写为向量检索友好的文本，多路改写取最优召回。

低置信度保护

意图置信度低于阈值时，Prompt 自动切换为引导澄清模式，不强行检索避免噪声答案。

无关问题拒识

系统 Prompt 内置知识边界守卫，超出 KB 范围的问题礼貌拒绝并引导提问。

07 · Context Engineering

上下文工程：有限窗口的信息密度最大化

在模型 context window 约束内，精确控制每一层内容的来源、优先级与 token 预算

Context Window 构成

System Prompt
角色定义 · 行为约束 · 安全边界
~400 tok
Retrieved Chunks
Rerank top-K · 来源标注 · 去重
~2000 tok
Conversation History
最近 N 轮 · 按 token 预算裁剪
~1200 tok
Current Query
改写后问题 + 意图标签
~200 tok

工程决策要点

Chunks 优先级高于历史

知识检索结果固定占用最大预算；对话历史在超限时从最远轮次开始裁剪，保证知识准确性。

t_message 持久化思考链

thinking_content / thinking_duration 字段持久化深度思考过程，可供后续评测与调试。

Answer Sources 溯源

sources_json 记录每条回答引用的原始文档 chunk，前端可展示引用来源面板。

KB 级记忆隔离

对话 session 绑定 kb_id，切换知识空间时强制 reset，彻底杜绝跨空间上下文污染。

08 · RAG Evaluation

RAGAS 评测体系：量化问答质量

黄金数据集 + 自动化评分，覆盖检索与生成两个维度

评测架构

EvaluationCollector

在 RAG 链路中通过 RagTraceContext 自动采集 query-chunk-answer 三元组，写入 t_rag_evaluation_record。

黄金数据集管理

4 张评测表（gold_dataset / gold_item / eval_run / eval_result），支持人工标注、批次运行、版本对比。

导出 & 自动评分

GET /rag/evaluations/export 产出 RAGAS-compatible JSON，run_eval.py 调用百炼 API 自动评分。

持续评测目标

每次 RAG 参数调整（改写策略、Chunk 大小、Rerank 模型）后可对比评测分数，形成量化迭代闭环。
