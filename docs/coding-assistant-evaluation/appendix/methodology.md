# 评测方法、评分口径与信息来源

> 截至 2026-04-29

## 1. 评测方法

本报告不进行三款工具的并行受控实测，而是依赖以下四类信息来源综合研判：厂商一手官方文档与企业版条款；独立技术分析机构和社区 benchmark（均标注发布日期）；厂商公开声称的能力描述；以及个别已公开的用户使用反馈。

三款候选工具（GitHub Copilot Enterprise、Gemini Code Assist、AWS Kiro）均为快速迭代的 SaaS 产品，能力矩阵在半年内可能发生显著变化。特别是 AWS Kiro 于 2025-07-17 进入 Preview，2025-11-17 正式 GA（基于 Code OSS 的独立 IDE），截至本报告评估基准日（2026-04-29）仅 GA 约 5 个月，区域覆盖和产品成熟度仍较有限。鉴于上述局限性，本报告仅代表截至 2026-04-29 的能力快照，使用者在采购决策前应自行核查最新文档。

## 2. 评分口径

本报告统一采用**三档定性评级：强 / 一般 / 弱**。每条评级均附理由与论据来源。不使用数字分（例如 1–5 分），原因如下：

1. **数字分易被误用**：各部门倾向于直接按总分排序，忽视分项差异和场景权重，容易导致选型失焦。
2. **三家企业版差异更适合质性表述**：三款工具在定位、部署形态和目标用户上存在本质差异，同一维度的数字分并不能反映适配性差异。
3. **无完全控制变量的实测条件**：本报告依赖文档和声称能力，非受控实验，打出数字精确分数是伪精确，不诚实。

对于无法从公开材料中核实的声明，统一标注为"未在公开材料中明确说明"，不进行推断或猜测。对于已识别的信息缺口，在相应位置显式标注并汇总于附录 vendor-terms.md 第 4 节。

## 3. 信息来源优先级

本报告按以下三档优先级使用信息来源：

**P0 一手（最高优先级）**：厂商官方文档、企业版条款页、定价页、产品博客（官方发布）、API/SDK 文档。具体包括：GitHub Docs（docs.github.com）、GitHub Blog、Google Cloud Documentation（docs.cloud.google.com）、Gemini Code Assist 数据治理页（developers.google.com/gemini-code-assist/docs/data-governance）、AWS Kiro 官网（kiro.dev）及其定价、企业版、区域支持页面。此类来源直接引用原文或 URL。

**P1 二手（次级参考）**：独立技术分析文章、社区 benchmark 平台（如 benchlm.ai、tbench.ai）、经过核验的非官方价格汇总页。使用时必须标注发布或数据更新日期，避免引用 6 个月以上的过旧数据。

**P2 基准对照（背景参考）**：Claude Code 官方文档（code.claude.com）、Anthropic 公开材料；OpenAI Codex 定价与 ChatGPT 订阅文档。此类来源用于提供行业能力基准，而非直接参与三款候选工具的横向比较。

当 P1 信息与 P0 信息冲突时，以 P0 为准；P2 仅用于建立背景认知，不用于候选工具的能力评级。

## 4. 时效性声明

本报告评估基准日为 **2026-04-29**。三款候选工具的迭代节奏均较快：GitHub Copilot 于 2026-03 月将 Agent Mode 和 Code Review GA，并于 2026-06-01 起切换为 AI Credits token 计费模式；Gemini Code Assist 正在扩展 Agent Mode 与私有代码定制能力；AWS Kiro 自 GA 仅约 5 个月，定价、区域覆盖和企业级合规功能仍在持续演进。

建议每 **6 个月**对本报告的能力矩阵和条款摘录进行一次复审，重点核对：各工具的定价结构、数据训练政策、新增合规认证，以及 Kiro 企业版功能的完善情况。
